# Answer the following question
## 為什麼加了 cache,還是會偶爾打到 DB?
* 因為 Cache 目的為降低 DB 壓力，並非完全取代 DB，因此打到 DB 是正常的。
* 常見原因如下：
1. **Cache Miss / Caching Pentration**: 第一次查詢，Key 不存在，TTL (Time-To-Live) 到期，Cache Service 重啟
2. **Cache stampede**: 同一個熱門 key 過期/大量存取，有可能發生 Cache Miss，導致 Request 全部直打 Database

## 50K QPS 全打到單一 PostgreSQL 會怎樣?
* 先延遲(latency)上升，然後錯誤率上升，最後整個 Service 可能會 Crash / Timeout
1. CPU, IO, Connection Pool 先飽和
2. p95/p99 Latency 快速上升
    * User 開始感覺到停頓
    * p95：第 95 位 User
    * p99：第 99 位 User
3. Race Competition, Lock
    * DB 由於會使用 Index 等其他機制加速查詢，導致
4. Application 層資源耗盡
